# 智慧餐配台（smart-catering-board）

一个给餐饮活动筹备人员使用的**纯前端工具**：
你可以配置「用料 → 菜品 → 菜单 → 项目桌型」，系统会自动计算整个项目所需总用料，并可导出整洁的 PNG 汇总图给客户或后厨。

## 1. 这个工具能做什么？
- 管理用料（名称、单位、备注）
- 管理菜品（可绑定多个用料和数量）
- 管理菜单（可绑定多个菜品和份数）
- 管理项目（多桌型堆叠、每桌可加菜/加用料）
- 自动汇总总用料 + 按桌明细 + 按菜品统计
- 一键导出汇总图（PNG）
- JSON 导入/导出备份（方便迁移电脑）

> 本工具不需要登录、不依赖后端，数据存储在浏览器本地。

## 2. 在线使用方式
部署到 GitHub Pages 后即可在线访问：
- 示例地址：`https://<你的GitHub用户名>.github.io/smart-catering-board/`

## 3. 本地启动（适合第一次体验）
1. 安装 Node.js（建议 20+）
2. 在项目目录执行：

```bash
npm install
npm run dev
```

3. 浏览器打开终端显示的地址（通常是 http://localhost:5173 ）

## 4. 使用流程（非技术同学版）
### 第一步：维护用料
进入「用料管理」，新增所有基础用料，比如：大米、鸡腿肉、食用油。

### 第二步：维护菜品
进入「菜品管理」，给每道菜配置用料和数量。
例如：宫保鸡丁 = 鸡腿肉 0.8 斤 + 食用油 0.08 瓶。

### 第三步：维护菜单
进入「菜单管理」，把多个菜品组合成一套菜单，并设置每道菜默认份数。

### 第四步：创建项目
进入「项目管理」→ 新建项目。
为项目添加桌型：
- 桌名称
- 菜单
- 这类桌有几围（堆叠）
- 额外加菜（按桌加）
- 额外用料（按桌加）

### 第五步：查看汇总并导图
项目列表里点「汇总」，可看到：
- 总用料汇总
- 按菜品统计
- 按桌型明细

点击「导出 PNG 图片」即可生成可直接发客户的图。

## 5. 导入 / 导出 JSON 数据
左侧栏有两个按钮：
- **导出 JSON 备份**：下载所有数据
- **导入 JSON**：把备份文件恢复到当前浏览器

建议每次大活动前后都备份一次。

## 6. 常见问题
### Q1：换电脑后数据不见了？
因为数据保存在浏览器本地，请先在旧电脑导出 JSON，再到新电脑导入。

### Q2：为什么我删了用料，菜品里还可能引用它？
当前版本为轻量实现，删除后会提示你手工检查关联菜品。

### Q3：可以多人协作吗？
当前版本定位单人使用，不提供账号和云端同步。

## 7. GitHub Pages 部署说明
本仓库已提供 `.github/workflows/deploy.yml`，推送到 `main` 后会自动部署。

你需要在 GitHub 仓库设置中确认：
1. `Settings` → `Pages`
2. `Build and deployment` 选择 **GitHub Actions**

然后每次 push 到 main 都会触发部署。

## 8. 项目结构（简版）
```
src/
  components/        # 通用界面组件
  pages/             # 页面模块
  lib/               # 存储、计算、ID 生成等逻辑
  state/             # 全局数据状态
  data/              # 默认示例数据
  types/             # TypeScript 类型定义
```
